## What is Cursor?

**Cursor is an AI-powered code editor** – think of it like a smarter version of Visual Studio Code (VS Code). It helps you write, understand, and fix code using artificial intelligence.

Think of it as having a super-smart coding assistant built right into your editor.

---

## What Can You Do With Cursor?

The documentation outlines 6 main things you can do:

1. **Understand your code** – If you open a big project (like someone else's code or a complex repository), Cursor helps you figure out how everything fits together. It traces connections so you know where to start making changes.

2. **Plan and build features** – You can describe what you want to build, and Cursor helps you plan the steps and write the code.

3. **Find and fix bugs** – Instead of spending hours searching for why something is broken, Cursor helps reproduce the issue, find the root cause, and suggest fixes.

4. **Review changes** – Before you merge your code with the main project, Cursor can inspect it, run checks, and catch problems early.

5. **Customize Cursor** – You can set up rules, skills, and prompts that match how you and your team work.

6. **Connect your workflow** – Cursor works with tools you already use like:
   - GitHub & GitLab (for code storage)
   - JetBrains (another code editor)
   - Slack & Linear (for team communication and project tracking)

## Models Explained (The Table)

The big table might look intimidating, but it's simple. **"Models"** are different AI brains that power Cursor. Each has slightly different strengths.

### What the columns mean:

| Column | What it means |
|--------|----------------|
| **Name** | The specific AI model (like Claude 4 Sonnet, GPT-5, etc.) |
| **Default Context** | How much code the AI can "see" at once (200k = ~150,000 words of code) |
| **Max Model** | The absolute maximum context it can handle |
| **Capabilities** | What the model can do (∞ symbol means "unlimited" or "full capabilities") |

### The Models (simplified):

- **Claude models** (from Anthropic) – Great for coding, reasoning, and following instructions
- **Gemini models** (from Google) – Fast and good with large amounts of text
- **GPT models** (from OpenAI) – The famous ChatGPT family, optimized for code
- **Composer** – Cursor's own specialized models
- **Grok** (from xAI) – Newer model, good for real-time information
- **Kimi K2.5** – Another specialized model

### Real-world example:
If you have a 500,000-character code file, you'd want a model with a "Max" of **1M** (1 million) so it can see your whole file at once.
