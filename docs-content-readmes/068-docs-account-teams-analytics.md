## Tracking AI Code in Git Commits (On-Device)

Cursor keeps a log of the signature of every AI line (Tab or Agent) suggested.

**How it works:**
1. AI suggestions are logged with unique signatures
2. Later, git commits from the same author are compared
3. Cursor detects line changes (additions/deletions) written by AI
4. Lines are attributed as AI-written

> *"All the AI detection is done on device, and never leaves the user's computer. We store the line counts as metadata."*

### Known Limitations:

| Limitation | Description |
|------------|-------------|
| **Diff signatures** | May be invalidated by automated code formatting |
| **Not implemented for** | Background Agents, Cursor CLI |
| **On-device requirement** | Git commit must be scored on same machine where AI code was authored |

## Analytics Charts & Metrics

### 1. AI Share of Committed Code

Shows lines of code changed in commits and what % was generated by Cursor.

| Definition | Meaning |
|------------|---------|
| **Cursor AI** | Any line attributable to Cursor Agent or Tab |
| **Other** | Line not detected as written by Cursor |

**Production branch filtering:** Uses default branch set in git repo, or falls back to `main`, `master`, `production`, `prod`.

---

### 2. Agent Edits

Shows amount of code edited by Agent and Cmd+K, and whether changes were accepted.

Group by:
- Suggested / Accepted
- File extension

---

### 3. Tab Completions

Shows number of times Tab code was suggested and accepted.

> *"Unit is Tab suggestions, regardless of lines of code changed in that suggestion."*

**Lines of code by Tab** is available via Analytics API.

---

### 4. Messages Sent

Shows number of messages sent by users to Cursor.

Filter by:
- Mode (Agent, Ask, Cmd+K)
- Model used

---

### 5. Active Users

Shows number of unique active Cursor users across different products.

**Definition:** User is active if they use at least one AI feature (Tab, Agent, Background Agent, CLI).

> *"Bugbot users are synced to GitHub accounts (not Cursor) and therefore not included in the all active user rollup."*

---

### 6. Daily Usage

Visualizes Cursor activity over preceding 365 days.

Toggle views:
| View | Shows |
|------|-------|
| **All** | Lines of code edits suggested by AI (Tab + Agent) |
| **Tab** | Number of suggestions made by Tab |
| **Agent** | Lines of code suggested by Agent |
| **DAU** | Daily active users across all Cursor products |

> *"Data collection for this chart starts in early September for customers on the 1.5+ desktop release."*

---

### 7. Usage Leaderboard

Shows top Cursor users across your team.

| Metric | Description |
|--------|-------------|

| **Tab Completions** | Number of Tab suggestions accepted |

### 8. Repository Insights

See how Cursor is used across different repositories.

| Metric | Description |
|--------|-------------|
| **AI Lines of Code Committed** | Code written by Cursor (Tab + Agent) and committed |
| **Total Lines of Code Committed** | All code committed by users |
| **Code Committed by AI %** | Percentage of committed code edited by Cursor |

> *"Some commits will be associated with Unknown repository if the user makes commits to a local git repository that doesn't contain a remote origin."*

### 9. Conversation Insights (Enterprise)

Cursor analyzes the code and context in each agent session to understand what kind of work is occurring.

> *"Teams no longer need high-toil, lossy analysis of tickets or low-response surveys to understand engineering work."*

**Classification Dimensions:**

| Dimension | Categories |
|-----------|------------|
| **Category** | Bug Fixing & Debugging, Code Refactoring, Code Explanation, Configuration, New Features, UI/Styling, Architecture, Data/Database, Documentation, DevOps/Deployment, Learning, Testing |
| **Work Type** | Maintenance (KTLO), Bug Fixing, New Features |
| **Complexity** | Distinguishes task complexity |
| **Specificity** | Measures how specific prompts are |

> *"Enterprise customers can extend these default categories or define their own across the organization or within specific teams."*

**Privacy:** All classification runs on-device. No PII or sensitive data leaves the machine.

**Pricing:** Free during preview period. Starting January 1, 2026, customers will be charged for inference plus Cursor Token Rate.

## Common Beginner Questions

### Q: Can regular team members see analytics?
**A:** Members can see their own data and the leaderboard, but not full team analytics (admin only).

### Q: How does Cursor know which code was written by AI?
**A:** On-device signature tracking – compares AI suggestion signatures to git commit diffs.

### Q: Does AI code tracking send my code to Cursor?
**A:** No – all detection is on-device. Only line counts (metadata) are stored.

### Q: What is Conversation Insights?
**A:** AI-powered classification of agent sessions into categories (bug fixing, refactoring, new features, etc.). Enterprise only.

### Q: Can I export analytics data?
**A:** Yes – CSV download per chart or bulk download. Enterprise has API access.

### Q: What versions of Cursor are my team using?
**A:** Check the Client Versions chart in analytics.

## The Bottom Line

**Usage Analytics gives team admins deep visibility into how Cursor is being used across their organization.**

**Key metrics to track:**
- **AI adoption** – % of code written by AI (AI Share of Committed Code)
- **User engagement** – Active Users, Usage Leaderboard
- **Activity by type** – Agent vs Tab vs Messages
- **Repository insights** – Which repos use AI most

- **Conversation Insights** (Enterprise) – What types of work are being done

**For team admins:** Use these analytics to understand adoption patterns, identify power users, and make data-driven decisions about AI tooling.
