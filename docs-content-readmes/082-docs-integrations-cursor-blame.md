---

## Overview

**Cursor Blame extends traditional git blame with AI attribution, so you can see what was AI-generated versus human-written.** Available on the Enterprise plan, it helps you understand the origin of each line when you review or revisit code.

### Attribution Types

| Type | Description |
|------|-------------|
| **Tab** | Code generated or accepted from Cursor's Tab suggestions |
| **Agent** | Code generated by Agent, with model attribution |
| **Human** | Code written directly by developers |

> *"Hover over AI-attributed lines to see conversation summaries. Click to open the full commit view with detailed AI attribution."*

---

## How It Works

Cursor Blame analyzes committed code to determine AI attribution. When you view a file or commit, it retrieves attribution data that identifies:

| Information |
|-------------|
| Which lines came from AI assistance |
| Which models were used |
| The conversations that produced them |

### Features

| Feature | Description |
|---------|-------------|
| **AI attribution** | See at a glance which lines came from AI vs manual |
| **Model tracking** | Agent-generated code shows which model produced it |
| **Conversation context** | View summaries of conversations that produced AI-generated code, with links to full commit details |
| **Contribution breakdown** | See percentage attribution for each contributor (AI models and human) in the commit view |

## Using Cursor Blame

### Enabling Cursor Blame

> *"Cursor Blame is disabled at the team level by default. A team admin can enable it for all team members in the Team settings."*

### Line Annotations (Ghost Text)

1. Open Command Palette (`Ctrl+Shift+P`)
2. Search for **"Cursor Blame: Toggle editor decorations"**

**Ghost text displays:**

| Field | Description |
|-------|-------------|
| **Author** | The team member who committed the change |
| **Subject** | The commit message |
| **Time** | When the change was made (relative time) |

### Hover Over Ghost Text

| Information |
|-------------|
| Author with AI co-authorship indicator |
| Conversation summary (for AI-attributed lines) |
| Commit statistics (files, additions, deletions) |

### Commit View

When you open the commit view, you see:

| Information |
|-------------|
| Full AI contribution breakdown by source |
| Percentage attribution for Composer, each Agent model, and human edits |
| Related conversations with summaries |

### File Blame View

| Step | Action |
|------|--------|
| 1 | Open a file in the editor |
| 2 | Right-click and select **Cursor Blame > Toggle file blame** (or Command Palette → "Cursor Blame: Open File Blame") |
| 3 | The file blame view shows attribution alongside each line |

## Common Beginner Questions

### Q: Is Cursor Blame available on Teams plans?
**A:** No – Cursor Blame is only available on Enterprise plans.

### Q: Is Cursor Blame enabled by default?
**A:** No – a team admin must enable it in Team settings.

### Q: What attribution types are shown?
**A:** Tab (code from Tab suggestions), Agent (code from Agent, with model attribution), and Human (manually written).

### Q: Can I see which model generated the code?
**A:** Yes – Agent-generated code shows which model produced it.

### Q: Can I see conversation context for AI-generated code?
**A:** Yes – hover over AI-attributed lines to see conversation summaries.

### Q: Does Cursor Blame store full conversation history?
**A:** No – only brief summaries are retrieved on-demand. Full conversation history is not stored.

## The Bottom Line

**Cursor Blame extends traditional git blame with AI attribution – showing what was AI-generated (Tab or Agent) versus human-written.**

**Key capabilities:**
- **Line-level attribution** – see origin of each line (Tab, Agent, Human)
- **Model tracking** – which model generated Agent code
- **Conversation context** – summaries of AI conversations that produced code

**For enterprise teams:** Cursor Blame is essential for:
